多线程

发表于2023-12-08|更新于2023-12-15|学习笔记
|浏览量:

简介

​ 多线程是一种程序的执行方式，它允许一个程序同时执行多个线程。在C++中，线程是操作系统能够进行运算调度的最小单位。一个进程可以包含多个线程，这些线程共享进程的资源，如内存空间、文件描述符等。

​ 在Linux中线程是轻量级的进程，从用户空间来看，线程是独立的执行流，但在内核层面，线程是作为一种特殊的进程来对待的，每个线程都有自己的进程描述符（task_struct），这与普通进程类似。它包含了线程的各种状态信息，如运行状态、优先级等。

​ 在 Windows 操作系统中，采用的是基于内核对象的线程模型。线程是操作系统进行调度的基本单位，每个线程都有自己的线程上下文（包括程序计数器、寄存器等）。线程由内核对象（ETHREAD结构）来表示，这个内核对象包含了线程的各种属性，如线程 ID、优先级等。当创建一个线程时，Windows 会为其分配相应的系统资源，并且线程的调度是由操作系统内核来完成的。

一个文本编辑器进程可能有一个线程用于接收用户的输入，另一个线程用于在后台保存文件，还有一个线程用于检查拼写错误。这样可以提高程序的响应速度和整体性能。

多线程的优点

1、提高了程序性能和响应能力

​ 在多核处理器环境下，多线程可以充分利用多个核心。

一个复杂的科学计算程序，如矩阵乘法运算。如果使用单线程，只能在一个核心上依次计算矩阵元素。而使用多线程，可以将矩阵划分为多个子区域，每个线程负责计算一个子区域，多个线程同时在不同的核心上进行计算，从而大大缩短计算时间。

​ 对于一些包含多个独立任务的程序，多线程允许这些任务同时进行。

以一个网络服务器为例，它需要同时处理多个客户端的连接请求。如果只有一个线程，服务器必须依次处理每个请求，当某个请求处理时间较长时，其他客户端就需要等待。而使用多线程，服务器可以为每个客户端连接创建一个新线程，这些线程可以并发处理不同客户端的请求，提高服务器对客户端请求的响应速度

2、可以更好的利用系统的资源，避免资源浪费

​ 多个线程可以共享进程的资源，如内存空间、文件描述符等。

在一个图像编辑软件中，多个线程可以同时访问和处理同一张图像的不同部分。一个线程可能负责图像的缩放操作，另一个线程负责颜色调整，它们共享图像数据在内存中的存储空间，这样可以避免重复加载数据，提高资源利用率。

​ 当一个线程因为等待 I/O 操作（如读取文件、网络通信等）而阻塞时，其他线程可以继续利用 CPU 资源进行其他任务。

在一个下载工具中，一个线程在等待从网络下载数据时（这可能会因为网络速度等因素而阻塞），另一个线程可以负责将已经下载的数据写入磁盘，从而使整个系统资源得到更充分的利用。

多线程的缺点

1、相比于单线程会更加复杂，需要深入理解线程的生命周期、同步机制和共享资源的管理等概念。程序调试也更加困难，因为多个线程的执行顺序是不确定的

2、资源消耗问题。每个线程都有自己的栈空间，线程的创建会消耗一定的内存。对于大量创建线程的应用程序，这可能会导致内存占用过高。

3、同步和并发带来的风险。当多个线程相互等待对方释放资源时，就会发生死锁。多个线程同时访问和修改共享数据时，如果没有正确的同步机制，就会出现数据竞争。这可能会导致数据不一致或不可预测的结果。

创建线程

C语言

pthread_create

  • 第一个参数，thread，传入参数，无符号的长整型数，线程创建成功后，会将线程ID写入到这个指针指向的内存中
  • 第二个参数，attr，线程的属性，一般情况下使用默认属性，NULL
  • 第三个参数，start_routine，函数指针，创建出子线程的处理动作，在该函数中子线程执行
  • 第四个参数，arg，作为实参传递到start_routine指针指向的函数内部。
  • 返回值，成功创建返回0，失败返回对应的错误号

pthread_self

每个线程都有唯一的线程ID，ID的类型为pthread_t，是无符号的长整型数，调用函数可以得到当前线程的线程ID。

示例

编译

主线程执行期间创建出了子线程，主线程有CPU的时间片，在这个时间片内将代码执行完毕后就退出了。子线程创建出来后需要抢CPU时间片，抢不到就不能运行。当主线程退出后，虚拟地址空间被释放，子线程就一起被销毁了。但如果子线程退出了，主线程仍在运行，虚拟地址空间依旧存在。所以在没有特殊情况下，虚拟地址空间的生命周期和主线程是一样的，与子线程无关。

pthread_exit

在编写多线程程序时，如果想要线程退出，但是不会导致虚拟地址空间的释放，我们需要用到线程库中的线程退出函数，只需要调用该函数，当前线程就马上退出，并且不会影响到其他线程的运行，不管在子线程还是主线程中都可以使用。

  • 参数，retval，线程退出的时候，携带的数据，当前子线程的主线程会得到该数据，如果不使用指定为MULL

编译

pthread_join()

线程在退出时，内核资源主要由主线程回收，线程库中提供的线程回收函数为pthread_join()，这是阻塞函数，当还有子线程在运行时，调用该函数会阻塞，子线程退出函数解除阻塞进行资源的回收，函数调用一次只能回收一个线程，如果多个子线程需要回收，则需要循环调用该函数。

  • 第一个参数，thread，要被回收的子线程的线程ID
  • 第二个参数，retval，传出参数，这个地址存储了pthread_exit() 传递出的数据，如果不需要这个参数，可以指定为NULL
  • 返回值，回收成功返回0，回收错误返回对应的错误号

​ 线程退出的时候使用pthread_exit()的参数将数据传出，回收这个子线程的时候可以通过pthread_join()的第二个参数来接收子线程传递出的数据。

pthread_detack()

在某些情况下，程序中的主线程有属于自己的业务处理流程，如果让主线程负责子线程的资源回收，调用pthread_join()只要子线程不退出主线程就会一直被阻塞，主要线程的任务也就不能被执行了。

这种情况，线程库函数中为我们提供了**线程分离函数pthread_detack()**，调用这个函数后，指定的子线程就可以和主线程分离，当子线程退出的时候，其占用的内核资源就被系统的的其他进程接管并回收。分离之后，主线程使用pthread_join()就回收不到子线程资源了。

pthread_cancel()

线程取消函数，在特定情况下在一个线程中杀死另一个线程。在线程A中调用线程取消函数pthread_cancel，指定杀死线程B，这时候线程B是死不了的，然后在线程B中进程一次系统调用（从用户区切换到内核区），否则线程B可以一直运行。

  • 参数，要杀死的线程的线程ID
  • 返回值，成功返回0，失败返回对应的错误号

C++

​ C++11之前，没有对并发编程提供语言级别的支持，在编写可移至的并发程序时有诸多的不便。C++11中增加了线程以及线程相关的类，很方便地支持了并发编程，使得编写的多线程程序的可移植性得到了很大的提高。

​ C++11中的线程类为std::thread，基于这个类创建一个新的线程非常的简单，只需要提供线程函数或者函数对象即可，并且可以同时指定线程函数的参数。

构造函数

  • 默认构造函数，构造一个线程对象，在这个线程中不执行任何处理动作
  • 移动构造函数，将other的线程所有权转移给新的thread对象，other不再表示执行线程
  • 创建线程对象，在该线程中执行g中的业务逻辑，args是要传递给函数f的参数。
    • f可以是普通函数、类成员函数、匿名函数、仿函数
    • 可以是可调用对象包装器类型，也可以是使用绑定器绑定之后得到的类型（仿函数）
  • 拷贝构造函数被删除，不允许线程之间进行拷贝构造

get_id()

通过该函数获取线程ID

示例

​ 在主线程中依次创建出两个子线程，打印两个子线程的线程ID，最后主线程执行完毕就退出了。默认情况下，主线程销毁时会将与其关联的两个子线程也一并销毁，但是这时有可能子线程中的任务还没有执行完毕，最后也就得不到我们想要的结果了。

join()

连接一个线程，主动地等待线程的中止，当前线程会阻塞，在某个线程中通过子线程对象调用join()函数，调用这个函数的线程被阻塞，但是子线程对象中的任务函数会继续执行，当任务执行完毕之后join()会清理当前子线程中的相关资源然后返回，同时，调用该函数的线程解除阻塞继续向下执行。函数在哪个线程中被执行，那么函数就阻塞哪个线程。

如果要阻塞主线程的执行，只需要在主线程中通过子线程对象调用这个方法即可，当调用这个方法的子线程对象中的任务函数执行完毕之后，主线程的阻塞也就随之解除了。

detach()

detach()函数的作用是进行线程分离，分离主线程和创建出的子线程。在线程分离之后，主线程退出也会一并销毁创建出的所有子线程，在主线程退出之前，它可以脱离主线程继续独立的运行，任务执行完毕之后，这个子线程会自动释放自己占用的系统资源。

线程分离函数detach()不会阻塞线程，子线程和主线程分离之后，在主线程中就不能再对这个子线程做任何控制了，比如：通过join()阻塞主线程等待子线程中的任务执行完毕，或者调用get_id()获取子线程的线程ID。

joinable()

用于判断主线程和子线程是否处理关联（连接）状态，一般情况下，二者之间的关系处于关联状态，该函数返回一个布尔类型：

  • true，主线程和子线程之间有关联（连接）关系
  • false，主线程和子线程之间没有关联（连接）关系

示例

  • 在创建的子线程对象的时候，如果没有指定任务函数，那么子线程不会启动，主线程和这个子线程也不会进行连接
  • 在创建的子线程对象的时候，如果指定了任务函数，子线程启动并执行任务，主线程和这个子线程自动连接成功
  • 子线程调用了detach()函数之后，父子线程分离，同时二者的连接断开，调用joinable()返回false
  • 在子线程调用了join()函数，子线程中的任务函数继续执行，直到任务处理完毕，这时join()会清理（回收）当前子线程的相关资源，所以这个子线程和主线程的连接也就断开了，因此，调用join()之后再调用joinable()会返回false。

operator=

线程中的资源是不能被复制的，因此通过 = 操作符进行赋值操作时不会得到两个完全相同的对象。

如果other是一个右值，会进行资源所有权的转移，如果不是右值，禁止拷贝，该函数被删除。

hardware_concurrency()

静态方法，用于获取当前计算机的CPU核心数，根据这个结果在程序中创建出数量相等的线程，每个线程独自占有一个CPU核心，这些线程就不用分时复用CPU时间片，此时程序的并发效率是最高的。
