什么是 C++ 模板(template)

C++ 模板是一种泛型编程的工具，它允许程序员独立于任何特定类型的方式编写代码。简单来说，模板可以看作是一个代码生成器，它根据用户提供的类型或其他模板参数生成特定类型的代码。

例如，考虑一个简单的函数，它可以交换两个变量的值。如果没有模板，可能需要为不同类型（如 int、double、char 等）分别编写交换函数。但有了模板，就可以编写一个通用的交换函数，适用于多种类型。

模板也是静态多态

模板的优缺点

优点

  • 代码复用性高，模板允许编写通用的代码，这些代码可以适用于多种不同的数据类型。
  • 类型安全，C++ 模板在编译时会进行类型检查。编译器会根据模板实例化时使用的具体类型来检查代码的正确性。
  • 模板可以用于实现抽象数据类型和通用算法。以容器类为例，标准模板库（STL）中的vector、list、map等容器都是模板类。它们可以存储不同类型的数据，用户在使用时只需要指定数据类型即可。这使得代码更加模块化和易于维护，同时也提高了代码的抽象层次，隐藏了具体的数据存储和操作细节。
  • 泛型编程是一种编程范式，它强调编写独立于特定数据类型的算法。C++ 模板是实现泛型编程的关键工具。通过模板，可以编写适用于各种类型的算法，而不依赖于具体的类型。

缺点

  • 编译时间长，模板代码在编译时会进行实例化，对于复杂的模板或者大量使用模板的项目，编译器需要为每个模板实例生成对应的代码。这会导致编译时间显著增加。
  • 代码膨胀，由于模板会为每个使用的类型生成一份独立的代码，这可能会导致生成的目标代码体积增大。特别是当模板用于多种不同类型，且这些类型在大小或操作上有较大差异时。
